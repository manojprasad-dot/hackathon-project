# 🔬 Technical Specifications & Mathematical Formulations

This document provides a deep-dive mathematical and algorithmic analysis of the on-device inference pipeline, feature extraction metrics, and heuristic-statistical model fusion in **PhishGuard Edge AI**.

## 1. Risk Score Fusion Engine

To ensure robustness against adversarial URL patterns while maintaining standard baseline classification confidence, PhishGuard fuses machine learning model predictions with heuristic evaluations.

The composite risk score ($R_{\text{final}}$) is represented by:

$$R_{\text{final}} = \alpha \cdot P_{\text{ONNX}} + \beta \cdot \min\left(1.0, \sum_{i=1}^{n} w_i \cdot f_i\right)$$

Where:
*   $\alpha = 0.75$ represents the statistical weight of the trained Random Forest classifier.
*   $\beta = 0.25$ represents the weight of the heuristic validation logic.
*   $P_{\text{ONNX}} \in [0, 1]$ is the scalar output probability from the ONNX WebAssembly prediction model.
*   $w_i \in [0, 1]$ represents the specific weight associated with heuristic category $i$.
*   $f_i \in \{0, 1\}$ is the logical outcome of rule evaluation $i$.

### Fusion Threshold Zones
*   **Safe Zone ($R_{\text{final}} \le 0.30$)**: The domain exhibits standard features with no anomalous tags.
*   **Warning Zone ($0.30 < R_{\text{final}} \le 0.65$)**: Low-confidence ML output paired with minor heuristic violations.
*   **Danger Zone ($R_{\text{final}} > 0.65$)**: High-severity heuristic flags or high-probability ONNX predictions. Immediate redirection is triggered.

## 2. Preprocessing & Feature Extraction

### Shannon Entropy Formulation
Shannon Entropy is computed to identify high-entropy hostnames generated by Domain Generation Algorithms (DGA) or random string obfuscation techniques.

For a hostname string $S$, the entropy $H(S)$ is defined as:

$$H(S) = -\sum_{i=1}^{k} P(s_i) \log_2 P(s_i)$$

Where:
*   $k$ is the number of distinct alphanumeric characters present in the hostname.
*   $P(s_i)$ is the probability of occurrence of character $s_i$ in string $S$:

$$P(s_i) = \frac{\text{Count}(s_i)}{\text{Length}(S)}$$

*   *Threshold*: Hostnames with $H(S) \ge 3.8$ are flagged under the high-entropy rule, indicating randomized DGA domains.

## 3. Homograph Attack Vector Detection

Visual spoofing utilizing Internationalized Domain Names (IDNs) is parsed by matching the unicode character sequences against the standard Punycode prefix pattern:

$$\text{is\_punycode} = \text{Regex}\left(\text{"\^xn--"}\right)$$

In addition, character-mixing checks scan for homoglyphs using scripts across mixed blocks (e.g., Cyrillic characters rendering identical shapes to Latin equivalents):

$$\text{is\_mixed\_script} = \left(\exists c_a \in \mathcal{C}_{\text{Latin}} \land \exists c_b \in \mathcal{C}_{\text{Cyrillic}} \text{ within Hostname}\right)$$

## 4. Model Training & Compilation Pipeline

### RandomForest Classifier Specs
The classifier is an ensemble of $T$ decision trees where $T = 100$. During training, bootstrap samples are drawn with replacement to split nodes by evaluating the Gini impurity:

$$I_G(p) = 1 - \sum_{i=1}^{J} p_i^2$$

To compile this model for client-side WASM execution without exceeding the memory constraints of browser extensions, we perform depth pruning:

$$\text{MaxDepth} = 15$$

This drops the final model footprint to **`1.66 MB`** while maintaining a validation accuracy of **`99.56%`**.
